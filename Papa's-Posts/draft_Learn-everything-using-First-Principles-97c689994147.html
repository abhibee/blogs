Learn everything using First Principles

In one of my previous posts, I had said — that only reason to learn something should be to satiate your curiosity. This is an expansion on th concept. 

Learning is a very personal thing. Don’t try to be like someone or try to learn something like someone. Learning does not work that way. For any topic that you need to learn, the very first thing you should do(even before asking why you should learn that?) is to try the topic from the first principles — Remember you are unique and you need not be like ANYONE else. First go try building that curiosity to learn. Question basic tenets in the area from the first principles most curiously and try finding convincing answers. If you start finding answers, you automotically start moving to level two of connecting with the topic and curiosity and questions further build on it. I am going to do an experiment here to test if the above works. I will try and learn 3 topics one after another using first principles and see where it lands me. 

  1. Write a comprehensive views on how to build a successful 1 Billion $ SAAS company in 5 years. The central plan and the goals for each department. The company can do anything — just that the founnders need to have very good understanding of the problem they are solving, that the people facing the problem are plenty and ready to give you a share of their wallet if you solve their problem sustainably. 
  2. Write a flowchart for the usage of various data science algorithms
  3. Write a model for a well oiled Engineering team. 
  4. Write notes on if SAFE works?
  5. Write a flowchart for various techniques used to cleanup/wrangle data.
  6. Make sense of Charlier Munger’s mental models list. 
  7. Call out what makes sense and what does not make sense in the book “Writing Tools” by Roy Clark
  8. Write a flow chart on which database to use when
  9. Write a flow chart on which Data Engineering tools and techniques to use when. 
  10. Write a Mental Model on how to use AWS most cost effectively.
  11. Find out if an end to end Camunda application makes any sense.
  12. Find out if graph and timeline databases are of any use. Try labels as graphs.
  13. Find out if timeline databases are of any use.
  14. Coaching and Counselling — does it work? How it works?
  15. Algorithms and data sctructures — Depth in java.
  16. What problems is LeadSquared facing and what is the solution plan?
